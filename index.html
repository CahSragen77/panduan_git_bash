🚀 PANDUAN LENGKAP LARAGON TO GITHUB

Pengingat langkah demi langkah biar nggak lupa lagi. Semangat ngulik, Mas Bro!

1. Menghubungkan Banyak Folder Proyek Terpisah ke Laragon

Jika folder proyek Anda tersebar di folder laptop yang berbeda (di luar C:\laragon\www), gunakan fitur Symlink agar Laragon tidak mengunci satu folder saja:

  • Buka aplikasi utama Laragon, klik tombol Terminal (Cmder).
  • Buka File Explorer, cari folder proyek Anda, lalu salin alamat jalurnya (path).
  • Ketik perintah pembuat jalan pintas virtual ini di terminal:
linker link "D:\Jalur\Folder\Proyek\Kamu"
  • Klik kanan pada area kosong di panel Laragon, pilih Apache > Reload.
  • Proyek siap diakses di browser melalui alamat: http://nama-folder.test

2. Cara Cepat Tes index.php / Membuka Proyek

Untuk langsung menguji file PHP di browser langsung dari terminal Cmder:

  • Pastikan posisi terminal sudah berada di dalam folder proyek tersebut.
  • Ketik perintah pembuka otomatis (pastikan tidak typo poen):
open .
⚠️ Catatan Penting: Pastikan tombol Start All di Laragon sudah diklik sampai berubah menjadi tulisan Stop, jika tidak website akan memunculkan error "Site cannot be reached".

3. Mengatasi Gembok Keamanan Git (Dubious Ownership)

Jika Git menolak perintah karena mendeteksi hak akses folder yang berbeda (biasanya efek pindah OS/akun Windows), buka gemboknya dengan perintah pengecualian global ini:

git config --global --add safe.directory D:/laragon/www/nama-folder-kamu

4. Mengatur Identitas Git Pertama Kali

Jika muncul error "Author identity unknown", beri tahu Git siapa pemiliknya:

git config --global user.email "email_kamu@gmail.com"
git config --global user.name "NamaAkunGitHub"

5. Alur Mengunggah Proyek Baru ke GitHub (Misi Sukses)

Berikut urutan perintah sakti dari awal inisialisasi sampai masuk ke cabang utama main di GitHub:

Langkah Awal di Website GitHub:

  • Buat repositori baru di website GitHub, beri nama, lalu klik Create repository.
  • Salin tautan HTTPS repositorinya (contoh: https://github.com).

Langkah Eksekusi di Terminal Laragon:

Jalankan perintah ini berurutan satu per satu:

git init
git add .
git commit -m "unggah berkas Pertama Proyek"
git branch -M main
git remote add origin TAUTAN_HTTPS_YANG_KAMU_COPY
git push -u origin main --force
💡 Mengapa pakai --force di akhir? Kita gunakan perintah paksa ini agar file codingan PHP dari laptop langsung menimpa file pembuka bawaan GitHub (README) tanpa memicu bentrok riwayat sejarah yang berbeda.

6. Cara Update Kode Selanjutnya (Setelah Sukses Pertama)

Nanti kalau Anda melakukan perubahan kode/codingan baru lagi, cukup ketik 3 perintah pendek ini saja di terminal:

git add .
git commit -m "isi catatan perubahan barumu di sini"
git push
